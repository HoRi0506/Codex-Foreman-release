# v0.0.5-pre

> Pre-release notice: `v0.0.5-pre` is still a beta/pre-release train. Install or update, fully restart Codex CLI, then run `ccc check-install`.

## Theme

`v0.0.5-pre` focuses on captain-constrained harness behavior.

## Exposure Policy

- The public release repo keeps `share/skills/cap/SKILL.md` as a thin placeholder.
- `scripts/build-release-asset.sh` copies the authoritative `$cap` skill from the source repo into the packaged release asset.
- Public browsing of the release tree should not expose the full installable skill text.
- History rewrite remains deferred; the current release-repo history stays intact until an explicit release decision changes it.

## Release Scope

- no-mutation preflight before default run creation
- MCP as a supported control-plane surface
- compact status as the allowed-action contract
- captain action guard for direct mutation, finish, review skip, and fallback
- mandatory fan-in before finish or follow-up
- visible direct-fallback and degraded-path history
- reduced `$cap` prompt surface with runtime/MCP-owned durable policy
- generated-defaults version 5 with stale default upgrades, including `way`/tactician from `gpt-5.5` `medium` to `gpt-5.5` `high`
- unsatisfactory/needs-work fan-in now canonicalizes to bounded specialist follow-ups, and captain-side local repair stays out of the path when CCC can route repair or reassignment through a specialist
- review-cap status now routes the captain to `ccc_orchestrate` for a captain decision instead of asking for another review
- worker and review lifecycle updates are split into `subagent_lifecycle` and `review_lifecycle`
- token usage visibility now carries explicit unavailable reason/code when host custom subagents do not provide raw usage events

## Asset Scope

- Published asset: `ccc-0.0.5-pre-darwin-arm64.tar.gz`
- Published asset: `ccc-0.0.5-pre-linux-x86_64.tar.gz`
- Published asset: `ccc-0.0.5-pre-windows-x86_64.tar.gz`
- macOS arm64 is the locally tested path.
- Linux x86_64 and Windows x86_64 assets are provided for early testing and may still have platform-specific issues because they have not yet been fully verified on real Linux or Windows environments.
- Linux arm64, Windows arm64, and other platform assets are not published for this pre-release yet.
