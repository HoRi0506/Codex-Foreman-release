# v0.0.2

- records delegated worker exit diagnostics more honestly by persisting exit code, raw output size, and raw output preview into delegation failure state
- lets `ccc_orchestrate` collapse `await_fan_in` and apply `replan` or `resolve` in the same bounded call instead of forcing an extra collapse round-trip
- trims ordinary `$cap` status/activity polling guidance so the host flow emits fewer visible `Ran ccc` tool calls during normal captain progress
- wires generic companion tool routing from `ccc-config.toml` into real runtime assignment instead of leaving it as recommendation-only metadata
- routes lightweight filesystem, fetch, Context7, OpenAI docs, and git read work to `companion_reader` when the routing policy selects that owner
- routes git mutation work to `companion_operator` when the routing policy selects that owner
- applies companion role model, reasoning tier, and sandbox selection to the actual delegated `codex exec` launch path
- surfaces the selected companion role in persisted entry traces and auto-entry answer traces so operators can see which lightweight specialist was chosen
- refreshes the packaged `$cap` skill so captain can explicitly reassign companion reader or companion operator work in bounded repair flows
- syncs CCC-managed Codex custom agents from `ccc-config.toml` during `setup` so the same role/model policy can back future official subagent delegation
- surfaces custom-agent sync health in `ccc check-install` instead of treating custom agents as an unimplemented packaged surface
- adds compact CLI payloads for the ordinary `$cap` loop so captain can use `ccc start/status/subagent-update/orchestrate` without repeatedly ingesting full run state
- compacts the packaged `$cap` skill and generated custom agent prompts to reduce repeated token cost during frequent captain and specialist calls
- validates the OMO-style harness path with config sync, single specialist, sequential chain, companion routing, fallback reason-code, and compact-output smokes
- fixes compact subagent-update fallback visibility and compact orchestrate command templates found during the OMO harness smoke pass
