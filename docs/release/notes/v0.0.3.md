# v0.0.3

- documents active-request handling when a new `$cap` arrives during an active run or subagent, including merge/replan/reclaim guidance when host custom subagents cannot be forcibly canceled
- adds the default scout lane policy: 2 read-only lanes for broad or parallel investigation, max 4
- adds the default raider lane policy: 2 lanes for broad or multi-file mutation, max 4, while single-file or shared-scope mutation stays sequential
- clarifies that token output is best-effort and only shows totals or gauges when raw usage events are available, otherwise printing an unavailable reason
- documents the `ccc-config.toml` edit path with a copy-paste `ccc setup`, Codex CLI restart, and `ccc check-install` refresh flow
- strengthens generated raider instructions around module boundaries, helper extraction, avoiding giant functions, and avoiding unrelated rewrites
- tightens generated specialist prompts so agents keep context compact and preserve the operator's language for user-facing text
- extends routing signals for Korean, English, Japanese, and Chinese requests across read, review, planning, mutation, docs, filesystem, and git/gh operations
- expands default config with companion reader/operator roles and git/gh-oriented companion routing using `gpt-5.4-mini` / `high`
- treats `gh` as a first-class routed tool signal, including read vs mutation ownership split (`companion_reader` / `companion_operator`) and default `gh` tool routing entries
- backfills missing `companion_agents` and `tool_routing` defaults in existing `ccc-config.toml` files without overwriting user-defined values
- makes `ccc status --text` print an explicit token-unavailable reason even when no delegated-worker usage events exist
- extends quiet CLI lifecycle lines (`status/start/orchestrate/subagent-update`) with compact token usage or explicit unavailable reason codes
- updates generated specialist/task prompts to handle user-provided external paths safely: search/read/copy when sandbox-allowed, otherwise report exact blocked path and approval requirement
- adds multilingual README coverage and role model/reasoning tables for English, Korean, and Japanese
- updates the default high-value role models for ChatGPT-authenticated Codex to `gpt-5.5` (`captain`, `tactician`, `raider`, and `arbiter`), while keeping `scout`, `scribe`, and companion roles on `gpt-5.4-mini` and documenting `gpt-5.4` fallback when `gpt-5.5` is unavailable
- adds release hardening guidance around a minimal release repo, stripped release binaries, and pre-release sensitive-string scanning
