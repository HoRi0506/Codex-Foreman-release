# v0.0.4

## Draft Scope

- keeps token gauges visible in text and quiet lifecycle output, with explicit unavailable reasons when host custom subagents do not expose raw usage events
- exposes structured `token_usage_visibility.status` and `token_usage_visibility.unavailable_reason_code` in status/activity payloads so JSON consumers can distinguish unavailable usage from missing data
- keeps raw delegated-worker token totals, per-agent breakdowns, and stacked gauges when usage events are available
- enforces route-backed companion ownership for lightweight tool routes: git/gh reads select `companion_reader`, and git/gh mutations select `companion_operator` unless captain records an explicit fallback or degradation reason
- keeps registered custom subagents as the default execution path and blocks silent direct `codex exec` fallback while a matching custom subagent is available
- documents file-handle pressure handling, including terminal lifecycle recording, merge/reclaim through captain, and closing used host subagents before continuing
- keeps fresh generated `ccc-config.toml` defaults on `gpt-5.4-mini` mini roles at `variant = "high"` and `fast_mode = true` while preserving existing user-customized values during setup backfill
- expands setup/check-install visibility with `installSurfaceVisibility` for MCP registration, canonical config, packaged `$cap` skill, and CCC-managed custom agents, including setup action and restart requirement
- records the cross-platform asset matrix for `darwin-arm64`, `darwin-x86_64`, `linux-arm64`, `linux-x86_64`, `windows-x86_64`, and `windows-arm64`, with native macOS/Linux install via `install.sh` and native Windows install via `install.ps1`
- adds non-mutating release asset matrix and Windows install smoke verification through `scripts/verify-release-asset-matrix.sh` and `scripts/verify-windows-install-smoke.sh`
- surfaces captain-owned review policy, reviewer fan-in outcomes, review pass caps, active-run/worker/file-handle/token/OS-memory/CPU resource-pressure suppression, and terminal host-subagent cleanup without allowing reviewers to auto-accept or auto-reassign work
- persists captain dissatisfaction/intervention state and queues exactly one bounded same-worker amend or explicit reassignment follow-up when the retry/reassign budget allows it
- exposes `completion_discipline` for document/checklist-driven finish-to-end requests so captain-visible status distinguishes completion-required work from a single bounded checkpoint
- completes the v0.0.4 oversized Rust entrypoint modularization target by extracting orchestration attempt/state helpers, subagent-update state helpers, MCP tool schema/result helpers, MCP dispatch, and the embedded test module while preserving CLI/MCP contracts
- requires a bounded review gate for failed validation, failed verification, failed acceptance, unresolved acceptance, and failing-test signals
- upgrades stale generated defaults through a generated-defaults version marker while backing up config and preserving customized values
